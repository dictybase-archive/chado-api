Chado

Chado API is to manage data in chado database through HTTP resources. The resource data is modeled and managed according to JSON API.

Query parameters

All resources with GET requests supports the following query parameters to refine responses…

  • include, e.g. GET /publications/34?include=authors to include all authors with publication.

  • sparse fieldsets, e.g. GET /publications/34?fields[publications]=title,abstract to return only title and abstract fields.

All collection resources with GET requests also supports the following query paramaters…

  • filter

  • pagination, e.g. GET /publications?page[number]=2&page[size]=10 to fetch second page of 10 publications.

Create a new publication

A new publication resource can be created either by …

  • supplying an id: The value should be in data.attributes.pubid field.

  • not supplying an id: In this case one will be created by the server.

The example below represents the first case.

REQUEST

POST

/publications
Headers
Content-Type application/vnd.api+json
Accept application/vnd.api+json
application/vnd.api+json
BODY SCHEMA 
                          {
    "data": {
        "type": "publications",
        "attributes": {
            "doi": "10.1002/dvg.22867",
            "title": "dictyBase 2015: Expanding data and annotations in a new software environment",
            "abstract": "dictyBase is the model organism database ......",
            "journal": "Genesis",
            "year": "2015",
            "volume": "12",
            "pages": "765-80",
            "month": "june",
            "issn": "1526-968X",
            "pub_type": "journal_article",
            "source": "pubmed",
            "pubid": "26088819",
            "issue": "8",
            "status": "ppublish"
        }
    }
}

RESPONSE

Headers
Content-Type application/vnd.api+json
Location /publications/26088819
201 application/vnd.api+json
BODY SCHEMA 
                        {
    "data": {
        "links": {
            "self": "/publications/26088819"
